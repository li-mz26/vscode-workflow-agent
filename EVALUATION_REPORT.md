## 📝 发现的问题和优化需求

### 🔴 需要修复的问题

1. **Python 布尔值问题**
   - 问题: `enabled: True` 中的 `True` 是 Python 语法，不是 JSON
   - 修复: 改为 `true`

2. **Switch 节点动态输出**
   - 问题: Switch 节点的输出端口是动态的，基于 conditions 配置
   - 当前: 需要手动管理输出端口
   - 建议: 根据 conditions 自动生成输出端口

3. **Parallel/Merge 节点动态端口**
   - 问题: 同样存在动态端口管理问题
   - 建议: 根据 branches 配置自动生成

### 🟡 建议的优化

1. **变量模板系统**
   - 当前: 简单的 `{{variable}}` 替换
   - 建议: 支持复杂表达式 `{{ctx.input.metric}}`

2. **错误处理节点**
   - 缺少: 专门的错误处理/重试节点
   - 建议: 增加 `retry`、`catch` 节点类型

3. **子工作流支持**
   - 缺少: 调用其他工作流的能力
   - 建议: 增加 `subflow` 节点类型

4. **执行状态追踪**
   - 当前: 基础的状态管理
   - 建议: 更详细的执行日志和状态可视化

5. **WebSocket 实时更新**
   - 当前: 轮询方式
   - 建议: WebSocket 推送执行状态

6. **节点模板库**
   - 缺少: 预置的常用节点模板
   - 建议: 内置告警处理、数据处理等模板
